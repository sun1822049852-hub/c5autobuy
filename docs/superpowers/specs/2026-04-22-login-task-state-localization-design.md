# Login Task State Localization Design

## 背景

当前 C5 账号登录抽屉会把后端任务状态直接显示给用户。

这导致普通用户能看到内部状态码，例如：

- `pending`
- `starting_browser`
- `waiting_for_scan`
- `idle`

同时，登录抽屉和默认占位文案里还残留开发阶段话术，例如：

- `这一版先把账号与代理上下文拉起来，任务状态下一阶段接入。`
- `等待接入真实任务流`

这些内容不适合作为正式发行版用户界面。

## 目标

把 C5 账号登录链中“用户可见”的任务状态与提示文案收口为正式中文。

要求如下：

- 只改显示层，不改后端状态协议。
- 普通用户界面不再直接出现内部状态码。
- 同一套状态码映射要集中维护，便于后续快速定位和修改。
- `waiting_for_browser_close` 的正式中文固定为 `等待关闭登录窗口`。

## 不可改变项

- 不修改后端任务状态枚举和值。
- 不修改登录任务轮询或推送协议。
- 不修改扫码登录流程、冲突处理流程、保存账号流程。
- 不把诊断面板里的“原始细节”通道删除，只禁止在主展示位直接裸露内部状态码。

## 设计方案

### 1. 集中状态映射

新增一个前端专用的登录任务状态映射模块，作为唯一维护点。

职责：

- 接收原始状态码
- 返回用户可见中文
- 为登录抽屉与诊断面板复用

映射固定如下：

- `idle` -> `未开始`
- `pending` -> `已创建`
- `starting_browser` -> `正在启动浏览器`
- `waiting_for_scan` -> `等待扫码`
- `captured_login_info` -> `已获取登录信息`
- `waiting_for_browser_close` -> `等待关闭登录窗口`
- `saving_account` -> `正在保存登录状态`
- `running` -> `进行中`
- `succeeded` -> `登录完成`
- `success` -> `登录完成`
- `failed` -> `登录失败`
- `cancelled` -> `已取消`
- `conflict` -> `账号冲突，等待确认`

未命中映射时，返回 `状态更新中`，避免继续裸露内部码。

### 2. 登录抽屉显示规则

登录抽屉中的两处状态显示统一走映射：

1. 任务状态卡
   - 当前 `task?.state || "idle"` 改为中文结果

2. 事件列表
   - 若后端已给出 `event.message`，优先显示消息
   - 若 `event.message` 为空，则回退为中文状态映射
   - 不再直接回退到 `event.state`

### 3. 日志弹窗显示规则

账号中心日志弹窗中的登录任务 meta 也不能继续写原始状态码。

当前 `状态：${taskSnapshot.state}` 改为：

- `状态：登录完成`
- `状态：等待扫码`
- `状态：等待关闭登录窗口`

等对应中文结果。

### 4. 正式文案替换

登录抽屉副标题改为正式用户口径，不再出现“这一版”“下一阶段”等开发态措辞。

目标文案：

- `登录会打开浏览器，请按页面提示完成扫码。`

默认日志种子中的半成品提示也替换为正式中文：

- `等待接入真实任务流` 改为 `尚未发起登录任务`
- 保留“最近登录任务 / 最近错误 / 最近修改”结构，但去掉明显开发阶段表达

### 5. 诊断面板处理

诊断面板属于可见界面，也复用同一套状态映射：

- 任务行右上角状态显示中文
- 时间线左侧状态显示中文

保留现有错误高亮、原始细节展开等调试能力，不扩大到其它诊断模块。

## 影响范围

会改动：

- 登录抽屉组件
- 登录任务诊断组件
- 账号中心默认日志种子
- 账号中心登录任务日志 meta 生成逻辑
- 新增一个登录状态映射模块
- 对应 renderer 测试

不会改动：

- 后端登录任务状态生产逻辑
- 账号登录主流程
- 非登录任务的其它状态体系

## 验证

需要补和通过以下验证：

- 登录抽屉测试：
  - 任务状态卡显示中文
  - 事件列表在无 message 时显示中文
  - 不再出现 `idle / pending / waiting_for_scan / starting_browser`
- 日志弹窗测试：
  - `状态：succeeded` 改为 `状态：登录完成`
- 诊断面板测试：
  - 最近登录任务列表中的 `task.state` 显示中文
  - 时间线中的 `event.state` 显示中文
- 账号中心相关测试：
  - 初始日志占位不再显示 `等待接入真实任务流`
