# 配置管理页压缩改造设计

日期：2026-03-21

## 1. 目标

在当前 `app_desktop_web` 的查询页基础上，做一次“微创压缩版”重排，让它从“查询工作台”转为“配置管理”。

本次只做已经确认的页面压缩与交互修正，不扩写新的业务功能。

目标是三件事：

1. 页面文案从“查询系统 / 查询工作台”统一收束成“配置管理”。
2. 去掉上方大 Hero，减少页面空白，提高当前配置与商品编辑区的密度。
3. 重做左侧配置栏与右侧保存入口，让配置管理更接近紧凑工具栏心智。

## 2. 当前问题

当前页面存在以下与目标不符的点：

1. 顶部保留 `Query System / 查询工作台` Hero 区，占据了过多纵向空间。
2. 左侧配置栏头部使用完整按钮 `新建配置`，视觉体积偏大。
3. 配置删除入口常驻在每个配置项右侧，视觉噪音偏高。
4. 底部单独存在 `保存区`，使页面结构被人为拉长。
5. “当前配置”卡片高度偏大，信息密度不足。

## 3. 冻结决策

### 3.1 页面改名

冻结为：

- 侧边导航中的 `查询系统` 改为 `配置管理`
- 页面内部不再出现 `查询工作台`
- 页面主区域按“配置管理页”表达

### 3.2 顶部 Hero

冻结为：

- 删除整块大 Hero
- 不再显示 `Query System`
- 不再显示 `查询工作台`
- 不再显示其说明副文案

页面顶部只保留错误信息区与主编辑布局。

### 3.3 左侧配置栏

左侧仍保留配置列表，但栏头交互改为：

- 一个绿色 `+`
- 一个普通 `-`

其中：

- 点击 `+` 打开现有 `新建配置` dialog
- 点击顶部 `-` 不直接删除任何配置
- 顶部 `-` 的作用是切换“删除态”

删除态冻结为：

- 默认状态下，每个配置项右侧不显示删除按钮
- 进入删除态后，每个配置项右侧浮现一个小 `-`
- 点击某个配置右侧的小 `-`，才进入现有删除确认弹窗
- 再次点击顶部 `-`，退出删除态
- 删除成功后，自动退出删除态

### 3.4 右侧当前配置区

右侧继续保留当前配置区，但要显著压缩高度。

冻结为：

- 标题仍是 `当前配置`
- 配置名、状态文案、运行状态仍保留
- 卡片纵向留白明显压缩
- `添加商品` 仍保留在右侧工具区

### 3.5 保存入口

底部 `保存区` 完全移除。

保存入口冻结为：

- `保存当前配置` 移入 `当前配置` 栏头右侧
- 与 `添加商品` 同一排
- 做成紧凑工具栏，而不是页面底部独立卡片

因此页面结构变为：

1. 左侧配置栏
2. 右侧当前配置栏头工具区
3. 下方商品列表

不再是：

1. Hero
2. 左侧配置栏
3. 当前配置卡
4. 商品列表
5. 保存区

## 4. 组件调整范围

本次改动只应触达以下区域：

- `app_desktop_web/src/features/query-system/query_system_page.jsx`
- `app_desktop_web/src/features/query-system/components/query_config_nav.jsx`
- `app_desktop_web/src/features/query-system/components/query_workbench_header.jsx`
- `app_desktop_web/src/features/query-system/components/query_save_bar.jsx`
- `app_desktop_web/src/styles/app.css`

必要时可在 `use_query_system_page.js` 中增加很薄的一层 UI 状态：

- `isDeleteMode`
- `toggleDeleteMode`
- 删除成功后自动退出删除态

但不改动配置保存、商品新增、商品编辑、运行态 overlay 的业务语义。

## 5. 交互细节

### 5.1 新建配置

- 顶部绿色 `+` 点击后，继续复用现有 `新建配置` dialog
- `新建配置` dialog 文案与行为暂不重做

### 5.2 删除配置

- 顶部 `-` 为删除态开关
- 仅在删除态中，配置项右侧才显示小 `-`
- 删除仍然复用现有确认 dialog

### 5.3 保存配置

- 用户修改配置后，保存动作不再去页面底部
- 直接在右侧“当前配置”工具栏点 `保存当前配置`
- 保存状态文案仍应保留，但以紧凑方式并入当前配置区

## 6. 视觉方向

本次不重新发明风格，只沿用当前暗色体系做压缩：

- 更少的纵向 padding
- 更少的标题层级
- 工具动作集中到栏头
- 删除入口默认隐藏，仅在删除态出现

目标视觉感受是：

- 更像“配置管理器”
- 更像“连续工作区”
- 减少“堆叠卡片”的松散感

## 7. 不在本次范围

以下内容明确不做：

- 商品行交互重构
- 查询分配算法调整
- 新的商品卡视觉语言
- 添加商品流程重写
- 删除商品功能新增
- 购买页改版

## 8. 验收口径

完成后应满足：

1. 页面上不再出现 `查询工作台`
2. 侧边导航显示 `配置管理`
3. 上方大 Hero 消失
4. 左栏头部显示 `+ / -`
5. 只有在删除态中，配置项右侧才出现小 `-`
6. 页面底部不再有独立 `保存区`
7. `保存当前配置` 已并入右侧当前配置工具栏
8. 当前配置区列高明显降低
