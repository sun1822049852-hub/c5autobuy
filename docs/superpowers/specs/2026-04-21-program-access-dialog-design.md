# Program Access Dialog Design

## 背景

当前左侧顶部的程序账号区把登录、注册、找回密码整套表单直接塞进了侧栏。

这与当前产品目标不符：

- 左侧更适合承担“入口 + 当前状态摘要”
- 具体账号操作应该在屏幕中央弹出独立窗口
- 已登录场景下不应继续看到登录表单，而应直接查看当前账号状态

## 不可改变项

- 不修改会员语义与共享工作区语义。
- 不修改只读锁定/功能解锁的业务判断。
- 不修改 `/program-auth/*` 接口路径与调用方式。
- 不把程序账号入口改成独立页面或新导航项。

## 交互方案

### 左侧入口卡

左侧顶部保留一张可点击的程序账号入口卡。

卡片只显示最小状态：

- 未登录时显示 `未登录`
- 已登录时显示 `用户名`

卡片不再展示以下细节：

- 权限说明
- 远端模式说明
- 到期时间
- 运行状态
- 登录/注册/找回密码表单

### 居中弹窗

点击左侧入口卡后，在屏幕中央拉起独立弹窗。

弹窗分两种状态：

1. 未登录
   - 默认进入登录视图
   - 可切换到注册、找回密码
   - 登录表单只存在于弹窗中

2. 已登录
   - 不再显示登录表单
   - 只显示当前账号状态
   - 保留 `刷新状态`、`退出`

### 用户名回显

为保证程序重开后左侧仍能显示正确账号名，`program_access` summary 需要正式透传 `username`。

来源规则：

- 远端登录态：从已验证 entitlement snapshot 中读取 `username`
- 本地放行或未登录：`username` 为空

## 实现边界

- 允许新增一个程序账号弹窗组件，或在现有 `ProgramAccessSidebarCard` 内拆出入口层与弹窗层。
- 允许补充 `program_access` 前后端 summary 的 `username` 字段。
- 不进行无关视觉重构，不扩大到其它页面布局。

## 验证

- Renderer 测试锁定：
  - 左侧卡只显示 `未登录/用户名`
  - 点击卡片后才出现登录/注册/找回密码表单
  - 已登录时弹窗仅显示状态与 `刷新状态`/`退出`
- Backend/bootstrap 测试锁定：
  - `program_access` summary 支持 `username`
  - 未登录时 `username` 为空
