# 配置管理商品行内暂停状态设计

日期：2026-04-23

## 背景

当前“配置管理”页里的商品级手动暂停虽然已经有真实状态，但控件被放在商品行之外的独立工具位，且按钮文案长期固定为“手动暂停”。用户在浏览商品列表时，很难直接判断一件商品现在到底是暂停还是运行。

本次设计只解决“商品级暂停状态看不出来”和“控件脱离商品行”的问题，不改变后端 `manual_paused` 语义，也不改变页面统一保存后才正式生效的既有机制。

## 目标

- 把商品级暂停控件收回商品条目所在同一行。
- 让用户仅通过图形就能直接识别当前状态：
  - 红色三角形：已暂停
  - 绿色双竖线：运行中
- 保持当前“点击只改草稿，页面统一保存后才正式生效”的配置管理语义。
- 保持删除模式仍然复用同一控制位，不额外增加新的浮动工具区。

## 非目标

- 不改后端 `manual_paused` 字段与运行时状态生成逻辑。
- 不改商品行内其他字段的编辑方式。
- 不改 `market_price` 只读约束。
- 不改查询运行时、购买运行时或 `查询 -> 命中 -> 购买` 主链行为。

## 已确认约束

- 图标不能独立悬在商品行外，必须和商品名、市场价、扫货价、磨损、`new_api`、`fast_api`、`token` 同排。
- 不显示“已暂停 / 运行中”文字，界面只显示图形状态。
- 图形优先用 CSS 绘制，避免依赖字符字体导致跨机器显示不一致。
- 无障碍语义仍要保留，例如 `aria-label="切换手动暂停 AK-47 | Redline"`。

## 最终交互方案

### 商品行结构

商品列表表头与每一条商品行都扩展为八列：

- 商品名
- 市场价
- 扫货价
- 磨损
- `new_api`
- `fast_api`
- `token`
- 状态

其中“状态”列就是原先独立在行外的控制位，但现在进入商品行内部，与其他字段同排对齐。

### 状态表达

- `manual_paused = true` 时，状态位显示红色三角形。
- `manual_paused = false` 时，状态位显示绿色双竖线。
- 点击状态位时，仍然只切换当前页面草稿中的 `manual_paused`，不会立即正式生效。

### 删除模式

- 正常模式下，状态列显示图形状态控件。
- 删除模式下，状态列原地替换为 `-` 删除按钮。
- 退出删除模式后，恢复图形状态控件。

这样既保持现有“删除模式抢占控制位”的交互，也避免状态控件再独立跑到行外。

## 视觉与布局原则

- 状态列宽度固定，保证所有商品行对齐。
- 状态控件与其他字段一样放在商品行卡片内部，不再单独占用右侧工具列。
- 运行中图标使用绿色双竖线，暂停图标使用红色三角形；两者都保持按钮热区一致，避免点击区域抖动。
- hover / focus 态保留按钮反馈，但不再依赖文字高亮来传达状态。

## 验证要求

- 商品列表初始渲染时，未暂停商品显示绿色双竖线，已暂停商品显示红色三角形。
- 点击状态位后，对应商品的三个模式状态文案继续切换为 `手动暂停`，证明草稿状态仍在生效。
- 删除模式开启后，图形状态位消失，并由同一格位的 `-` 删除按钮替换。
- 页面保存行为、保存 payload 中的 `manual_paused` 字段，以及只读锁定态都保持不变。
