# 配置管理单项编辑弹窗设计

日期：2026-04-13

## 背景

当前“配置管理”页中，商品行内多个可编辑项都会汇入同一个大编辑弹窗。这个交互把不同修改目标混在一起，导致用户点击某个字段时，看到的是整条商品的全集编辑界面，而不是与当前动作严格对应的局部修改界面。

本次设计目标是把“点哪个改哪个”的心智模型固定下来，同时保留现有页面级统一保存机制，不引入每个弹窗各自的提交流程。

## 目标

- 每个可编辑项都有自己的单项编辑弹窗。
- 用户在弹窗中的修改直接写入当前页面草稿。
- 真正生效仍然只依赖页面现有的统一保存动作。
- `手动暂停` 作为配置修改的一部分，进入草稿但不即时正式生效。
- 删除模式占用 `手动暂停` 的按钮位，避免额外挤压行内布局。

## 非目标

- 不改变页面级保存机制。
- 不增加每个弹窗自己的“确认修改”或“取消回滚”提交层。
- 不允许手动修改 `market_price`。
- 不把多个字段合并成一个多项弹窗。

## 已确认约束

- `market_price` 来自自动查询，只展示，不可编辑。
- `扫货价`、`磨损`、`new_api`、`fast_api`、`token` 都必须各自独立，使用单项编辑弹窗。
- `手动暂停` 不开弹窗，点击后直接修改当前页面草稿，仍需页面统一保存后才正式生效。
- 关闭单项编辑弹窗时，不回滚已写入草稿的修改；弹窗只是关闭视图。

## 最终交互方案

### 商品行结构

商品行保留当前高密度表格布局，但重新定义点击行为：

- 商品名：只展示。
- `market_price`：只展示，不可点击。
- `扫货价`：点击打开“扫货价单项编辑弹窗”。
- `磨损`：点击打开“磨损单项编辑弹窗”。
- `new_api`：点击打开“new_api 单项编辑弹窗”。
- `fast_api`：点击打开“fast_api 单项编辑弹窗”。
- `token`：点击打开“token 单项编辑弹窗”。
- `手动暂停`：位于 `token` 右侧，作为单独启停控件。

### 删除模式

- 正常模式下，`token` 右侧显示 `手动暂停` 控件。
- 进入删除模式后，该位置临时替换为 `-` 删除按钮。
- 退出删除模式后恢复 `手动暂停` 控件。

这样删除动作与暂停动作共享同一控制位，不新增额外列，也不破坏现有行宽。

## 单项编辑弹窗定义

### 1. 扫货价单项编辑弹窗

- 标题：`修改扫货价`
- 内容：
  - 商品名
  - `market_price` 当前值，只读参考
  - `扫货价` 输入框
- 行为：
  - 输入变化后立即写入当前配置草稿
  - 不提供独立提交按钮

### 2. 磨损单项编辑弹窗

- 标题：`修改磨损`
- 内容：
  - 商品名
  - 天然磨损范围，只读参考
  - `最小磨损` 输入框
  - `最大磨损` 输入框
- 行为：
  - 输入变化后立即写入当前配置草稿
  - 不提供独立提交按钮

### 3. 模式单项编辑弹窗

三种模式完全对称，但互不合并：

- `修改 new_api 分配`
- `修改 fast_api 分配`
- `修改 token 分配`

每个弹窗只包含本模式自己的信息：

- 商品名
- 当前模式状态文案
- 当前模式可分配余量
- 当前模式分配数量输入框

行为：

- 输入变化后立即写入当前配置草稿
- 不出现其他模式字段
- 不承担最终保存责任

## 草稿与保存模型

本次设计明确采用“两层确认，但只有一层提交”的模型：

1. 单项编辑弹窗只负责聚焦当前字段或模式的编辑。
2. 所有输入变化直接进入页面草稿。
3. 页面现有统一保存按钮是唯一正式提交入口。

用户看到的结果应当是：

- 弹窗内修改后，列表立即反映草稿值。
- 页面出现“未保存”状态。
- 只有点击页面统一保存后，配置才正式生效。

## 组件改造方向

### 页面层

- 移除现有“整条商品大编辑弹窗”的唯一入口定位。
- 改成维护“当前打开的单项编辑上下文”。

### 行组件

- 为每个可编辑格子提供独立点击行为。
- `market_price` 由按钮改为纯展示。
- 新增 `手动暂停` 控件位，并支持删除模式替换。

### 编辑状态

- 现有统一的整条 `editItemDraft` 逻辑，重构为可被单项弹窗复用的草稿写入能力。
- 单项弹窗根据字段类型只读取和修改自己负责的那部分草稿。

## 验证要点

实现后至少验证以下行为：

- 点击 `market_price` 不会打开任何编辑界面。
- 点击 `扫货价`、`磨损`、三种模式时，分别只打开对应单项编辑弹窗。
- 模式弹窗内不会出现其他模式字段。
- `手动暂停` 点击后只改变草稿，不会绕过页面统一保存。
- 删除模式下，`手动暂停` 位置被 `-` 替换。
- 关闭单项编辑弹窗后，已改的草稿仍然保留。
- 页面统一保存后，所有草稿修改一起正式生效。
