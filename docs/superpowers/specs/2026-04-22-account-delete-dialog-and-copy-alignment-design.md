# 账号中心删号弹层与用户文案对齐设计

## 目标

把账号中心“删除账号”从浏览器原生确认框替换为项目现有 `dialog-surface` 风格的站内确认弹层，并顺手收口普通用户直接可见的英文眉标/开发感文案，让账号中心与程序账号入口的视觉口径更一致。

## 现状

- 账号中心右键菜单中的“删除账号”目前直接调用 `window.confirm(...)`，交互样式跳出当前产品风格。
- 普通用户可见界面仍存在几处英文眉标：
  - 账号中心页的 `ACCOUNT CENTER`
  - 程序账号入口与程序账号弹窗里的 `PROGRAM ACCESS`
  - 诊断面板里的 `Diagnostics`
- 本地放行调试分支中仍有“本地调试模式”提示，但它属于显式调试入口，不应与正式用户口径混为一谈。

## 设计决策

### 1. 删除账号确认交互

- 保持右键菜单中的“删除账号”入口不变。
- 点击后不再弹浏览器确认框，而是打开账号中心内的居中确认弹层。
- 弹层内容极简：
  - 标题：`删除账号`
  - 中间仅显示被删账号名
  - 底部按钮：`取消`、`确认删除`
- 不新增说明文字、不增加二次输入、不改变删除语义。
- 点击遮罩或“取消”关闭弹层；点击“确认删除”继续走现有删除接口和刷新链路。

### 2. 用户可见文案对齐

- 只改普通用户可直接看到的英文眉标，不做全站术语翻译。
- 拟统一为：
  - `ACCOUNT CENTER` -> `账号中心`
  - `PROGRAM ACCESS` -> `程序账号`
  - `Diagnostics` -> `运行诊断`
- 保留调试专用提示：
  - `本地调试模式` 只出现在 `local_pass_through` 分支，属于显式调试入口，不纳入本轮清理。

## 不改项

- 不改账号删除接口、删除权限、删除后刷新逻辑。
- 不改右键菜单位置与其它菜单动作。
- 不改本地调试入口的功能语义。
- 不做全局对话框系统抽象或样式重构。

## 验证策略

- Renderer 测试覆盖：
  - 右键菜单点击删除后出现站内弹层而不是 `window.confirm`
  - 弹层确认后发送 DELETE 请求并刷新列表
  - 账号中心页、程序账号入口、诊断页展示中文眉标
- 受影响页面做合并回归，防止 copy 调整带出旧断言失败。
