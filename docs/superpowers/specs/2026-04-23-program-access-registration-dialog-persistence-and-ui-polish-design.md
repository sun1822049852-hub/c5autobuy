# 程序账号注册弹窗状态保持与 UI 收口设计

## 目标

在不改远端注册接口、不改登录/找回密码业务链路的前提下，收口程序账号注册弹窗的两类问题：

- 注册第二段验证码页在关闭弹窗、切换主界面后重新打开时，不应要求用户重新填写邮箱或重新发验证码。
- 程序账号弹窗的默认提示与输入样式需要更接近当前产品要求，去掉无意义拦截提示，减少表单噪声。

## 不改项

- 不改 `send-code / verify-code / complete` 三接口契约。
- 不改 `registration_flow_version` 的 v2/v3 切换逻辑。
- 不改登录、找回密码、程序会员状态刷新与退出链路。
- 不改验证码冷却规则、验证码错误映射与第三段注册提交逻辑。
- 不做 renderer 刷新后的跨会话持久化，本轮只保证当前 renderer 会话内的状态保持。

## 现状问题

### 1. 关闭弹窗会把注册状态直接清空

当前程序账号弹窗执行关闭动作时，会无差别调用注册流重置逻辑并把模式切回 `login`。结果是：

- 用户已经完成第一段发码、正停留在第二段验证码页时，只要关闭弹窗再回来，就会被打回登录页。
- 重新点“注册”后又回到第一段邮箱页，必须重新输入邮箱。
- 前端本地倒计时也被清空，用户只能重新发验证码。

这与已确认的产品行为不符。

### 2. 默认拦截提示噪声过高

未登录状态下，弹窗顶部会直接显示 `program_auth_required / 请先登录程序会员`。这条提示不是异常，而是未登录时的默认状态，保留它只会制造额外视觉噪声。

### 3. 表单视觉过重

当前注册与登录表单仍以“外置字段标题 + 深色输入框 + 普通灰色关闭按钮”为主：

- 输入提示与字段标题重复。
- 输入区域背景偏暗，不利于聚焦正在填写的内容。
- 右上角关闭按钮不够明显，不符合用户要求的红底正方形 `X`。

## 设计决策

## 1. 注册第二段验证码页在当前 renderer 会话内保持

本轮采用用户确认的方案 A：只在当前 renderer 会话内保留注册草稿，不把状态写入 `localStorage/sessionStorage`。

需要保留的字段包括：

- `authMode=register`
- `registerStep=register_code`
- `registerForm.email`
- `registerForm.code`
- `registerSessionId`
- `maskedRegisterEmail`
- `registerResendCooldownSeconds`

关闭弹窗、点击遮罩关闭、切换主导航页面后，这些值都不应被主动清空。用户重新打开程序账号弹窗时：

- 若此前停留在第二段验证码页，则直接回到第二段。
- 邮箱摘要和倒计时继续显示。
- 不自动重发验证码。

## 2. 哪些动作会清空注册草稿

以下场景仍允许清空或重置注册流：

- 用户主动点击“修改邮箱”
- 用户主动从“注册”切到“登录”或“找回密码”
- 注册成功并完成流程
- 第三段因 `REGISTER_TICKET_INVALID_OR_EXPIRED` 被强制退回第二段时，按既有逻辑仅清空账号名/密码与票据，不清空邮箱会话

换言之，本轮把“关闭弹窗/切页面”从“重置注册流程”中剔除，但保留“主动离开注册模式”时的清理语义。

## 3. 隐藏默认 `program_auth_required` 顶部告警

顶部红色告警区只保留真正异常，不再显示“未登录即默认缺少程序会员”的提示。

具体规则：

- 当 `lastProgramAuthError.code === "program_auth_required"` 时，不显示顶部告警
- 当 `access.lastErrorCode === "program_auth_required"` 且没有其它更具体的 action error 时，不显示顶部告警
- 其它错误例如服务不可用、注册失败、验证码错误、网关异常仍照常显示

## 4. 输入提示改为 placeholder，保留无障碍标签

登录、注册、找回密码三类输入框统一收口为：

- 视觉上以输入框内 `placeholder` 提示为主
- 保留 `aria-label`，不破坏现有测试与可访问性
- 外置字段标题不再显示，减少弹窗纵向噪声

对应字段包括：

- 程序会员登录账号
- 程序会员登录密码
- 注册邮箱
- 注册验证码
- 注册用户名
- 注册密码
- 找回密码邮箱
- 找回密码验证码
- 新密码

## 5. 输入框与关闭按钮样式收口

### 输入框

- 输入框背景改为浅色背景，而不是当前偏黑底
- 占位提示文字使用浅灰但可读的颜色
- 保持现有圆角与尺寸，不重做布局

### 关闭按钮

- 右上角关闭按钮改为红底正方形 `X`
- 按钮可访问名称仍为“关闭”
- 仅收口视觉样式，不改点击行为

## 测试要求

前端测试至少覆盖以下行为：

- 注册 v3 发码成功进入第二段后，关闭弹窗再打开，仍停留在第二段验证码页，并保留脱敏邮箱与倒计时
- 用户主动切到“登录”标签后，注册草稿被清空，再回“注册”时回到第一段邮箱页
- `program_auth_required / 请先登录程序会员` 不再显示
- 输入框具备对应 placeholder
- 关闭按钮仍可通过“关闭”访问名命中，同时视觉文本为 `X`

## 成功判定

满足以下条件即视为本轮设计落地正确：

- 用户在第二段验证码页关闭弹窗或切换页面后，再回来无需重新填邮箱、无需重新发验证码。
- 倒计时连续，不会因为关闭弹窗而重置。
- 顶部不再展示默认 `program_auth_required / 请先登录程序会员` 提示。
- 表单输入提示进入输入框内部，输入框视觉更轻，关闭按钮为红底正方形 `X`。
